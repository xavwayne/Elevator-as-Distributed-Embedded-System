Portfolio Overview

  • Design
    • Architecture Diagram - The architecture diagram describes the objects present in the system, the replication of distributed objects, and the communication channels between objects.
    • Use Cases - The use case diagram describes the ways that agents in the system can interact with the elevator and shows relationships between these methods of interaction.
    • Scenarios and Sequence Diagrams - Scenarios describe user interaction with the system.  Sequence diagrams describe the scenario events in greater detail, including the message interaction between various system componenets.
    • Requirements I - System Object Descriptions and Message Dictionary - A list of the sensors and actuators in the system and a message dictionary describing the messages that are required for the various controllers.
    • Requirements II - Distributed Controller Requirements - Detailed specifications for the all controllers in the system.
  • Traceability
    • Sequence Diagrams to Requirements Traceability - Forward and backward traceability from sequence diagrams to event triggered requirements from Proj3, then to time triggered requirements from Proj4.
    • Requirements to Contstraints Traceability - Forward and backward traceability between the requirements behaviors and the constraints given for each module.
    • Statecharts to Code Traceability - Shows where each transition described in the controller statecharts is implemented in the code.
  • Implementation
    • Elevator Control Package - Lists and describes all java files in elevatorcontrol package.
  • Test
    • Unit Test Log - Describes the unit tests and summarizes the current unit test results.
    • Unit Test Summary File - Parseable list of unit test files.
    • Integration Test Log - Describes the integration tests and summarizes the current integration test results.
    • Integration Test Summary File - Parseable list of integration test files.
    • Acceptance Test Log - Describes the accpetance tests and summarizes the current acceptance test results.
    • Fault Tolerance Test Log - Describes the fault tolerance tests and summarizes the current test results.
  • Log Files
    • Issue Log - Details for each design issue during the semester.
    • Improvements Log - List of ideas or suggestions for improving the project.
    • Peer Review Log - Details for each peer review event that is performed during the semester.
